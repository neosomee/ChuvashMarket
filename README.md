## 📦 Установка

### Требования

- Node.js 18+ и npm
- Python 3.8+ и pip
- Git

### Клонирование репозитория

```bash
git clone <repository-url>
cd Market
```

### Установка зависимостей frontend

```bash
npm install
```

Установятся:

- react@19.2.0
- react-dom@19.2.0
- react-router-dom@7.1.1
- recharts (для графиков)
- lucide-react (иконки)
- vite@7.2.4

### Установка зависимостей backend

```bash
cd backend
pip install -r requirements.txt
```

Установятся:

- Django
- djangorestframework
- django-cors-headers
- djangorestframework-simplejwt

## 🚀 Быстрый старт

### 1. Инициализация базы данных

```bash
cd backend
python manage.py migrate
python init_data.py  # Создает тестовые данные
```

Это создаст:

- **Суперпользователя**: `admin / admin123`
- **Продавцов**:
  - `seller_honey / seller123` (тестовый продавец с 5 товарами и 30+ заказами)
  - `seller_crafts / seller123`
  - `seller_farm / seller123`
- **Покупателей**: `buyer_anna / buyer123`, `buyer_oleg / buyer123` и других
- **10+ товаров** с реалистичными данными и ценами
- **60+ заказов** за последние 30 дней с различными статусами

### 2. Запуск backend сервера

```bash
cd backend
python manage.py runserver
```

Сервер запустится на `http://127.0.0.1:8000`

Backend endpoints:

- API: http://127.0.0.1:8000/api/
- Admin: http://127.0.0.1:8000/admin/

### 3. Запуск frontend dev сервера

```bash
# В отдельном терминале из корня проекта
npm run dev
```

Приложение откроется на `http://127.0.0.1:5173` с HMR

### 4. Вход в систему

**Как продавец (с полной аналитикой):**

1. Перейти в "Вход" или на `/auth/login`
2. Учетные данные: `seller_honey / seller123`
3. После входа в меню выбрать "Панель продавца"
4. Увидеть полную аналитику с 9 интерактивными графиками

**Как покупатель:**

1. Учетные данные: `buyer_anna / buyer123`
2. Пролистать каталог, добавить товары в корзину
3. Оформить заказ
4. Написать отзыв на товар

**Как администратор:**

1. Учетные данные: `admin / admin123`
2. Доступ к админ панели: http://127.0.0.1:8000/admin/

## 📊 Панель продавца - Детальная аналитика

### 9 Интерактивных компонентов

#### 1️⃣ Карточки базовых метрик (StatCard)

4 главные метрики для быстрого просмотра:

- 💰 **Выручка (30 дней)** с средним доходом
- 💳 **Средний чек** с количеством завершенных заказов
- 📦 **Единиц продано** с количеством активных товаров
- ✅ **Выполнение заказов** с процентом выполнения

Цвета: синий, зеленый, оранжевый, зеленый
Иконки: TrendingUp, ShoppingCart, Package, CheckCircle

#### 2️⃣ График дохода (RevenueChart)

Интерактивный линейный график с заливкой:

- 📈 Ежедневный доход за последние 30 дней
- 📊 Количество заказов в день
- 🎯 Подсказки при наведении курсора
- 📉 Статистика среднего дохода в день
- 💵 Общая выручка за период

Формат данных: [{date: "2025-02-01", revenue: 1000, orders: 5}, ...]

#### 3️⃣ Статус заказов (OrdersStatusChart)

Круговая диаграмма с двумя сегментами:

- ✅ **Завершенные** (зеленый)
- ⏱️ **Ожидающие** (оранжевый)
- 📊 Процент выполнения
- 🔢 Общее количество заказов

#### 4️⃣ Топ товаров (TopProductsChart)

Столбчатый график с двумя осями:

- **Левая ось**: Доход (синие столбцы)
- **Правая ось**: Количество проданных единиц (голубые столбцы)
- 📊 Показывает топ 8 товаров по выручке
- 🎯 Интерактивные подсказки при наведении

#### 5️⃣ Расширенная статистика (AdvancedStatsGrid)

Сетка из 6 дополнительных метрик:

- 👥 **Уникальные покупатели** - количество разных клиентов
- ⭐ **Средний рейтинг** - оценка товаров (0-5 звезд) с количеством отзывов
- 📦 **Средний размер заказа** - среднее количество единиц товара в заказе
- 📊 **Общий объем продаж** - всего проданных единиц
- 💰 **Средняя цена товара** - среднее значение в каталоге
- 📈 **Диапазон цен** - от минимума к максимуму

#### 6️⃣ Рейтинг товаров (RatingsChart)

Два списка товаров на основе отзывов:

- 🌟 **Лучшие товары** (топ-5 с наивысшим рейтингом, зеленый цвет)
- ⚠️ **Товары для улучшения** (топ-5 с наихудшим рейтингом, оранжевый цвет)
- Визуализация звезд для каждого товара
- Точная оценка (например, 4.5 ⭐)

#### 7️⃣ Статус товаров (PublishedStatusChart)

Столбчатый график распределения:

- 📤 **Опубликованы** (зеленый) - товары в каталоге
- 📥 **Черновики** (оранжевый) - неопубликованные товары
- 🔢 Количество в каждой категории

#### 8️⃣ Распределение цен (PriceRangesChart)

Круговая диаграмма с 4 ценовыми диапазонами:

- 💰 **Бюджет** (< 1K) - доступные товары - синий
- 💳 **Средний** (1K-5K) - большинство товаров - голубой
- 🎁 **Премиум** (5K-20K) - дорогие товары - светло-голубой
- 👑 **Люкс** (> 20K) - эксклюзивные товары - зеленый

#### 9️⃣ Доход по статусам заказов (OrderStatusRevenueChart)

Столбчатый график дохода по статусам:

- 🔵 **Новый** (pending) - серый
- 🟠 **Оплачен** (paid) - оранжевый
- 🟦 **Отправлен** (shipped) - синий
- 🟢 **Завершен** (completed) - зеленый
- 🔴 **Отменен** (canceled) - красный

### Параметры аналитики (20+ метрик)

**Базовые метрики:**

1. `products_count` - всего товаров
2. `published_count` - опубликованные
3. `draft_count` - черновики
4. `orders_count` - всего заказов
5. `pending_orders` - ожидающие

**Расширенные метрики товаров:** 6. `unique_buyers` - уникальные покупатели 7. `total_units_sold` - общий объем продаж 8. `avg_product_price` - средняя цена товара 9. `min_product_price` - минимальная цена 10. `max_product_price` - максимальная цена

**Метрики заказов:** 11. `total_revenue` - общая выручка за 30 дней 12. `avg_order_size` - средний размер заказа 13. `order_statuses` - распределение по 5 статусам 14. `revenue_by_status` - доход по статусам

**Метрики оценок:** 15. `reviews_count` - количество отзывов 16. `avg_rating` - средний рейтинг (0-5) 17. `best_products` - лучшие товары (топ-5) 18. `worst_products` - худшие товары (топ-5)

**Метрики диапазонов:** 19. `published_by_status` - распределение (опубликованы/черновики) 20. `price_ranges` - распределение по ценовым диапазонам

**Графики с данными:** 21. `top_products` - топ товаров с доходом и количеством 22. `sales_last_30` - ежедневные продажи за 30 дней

## 🔌 API Документация

### Authentication (JWT)

```
POST /api/auth/login/
Body: { "username": "...", "password": "..." }
Returns: { "access": "JWT_TOKEN", "refresh": "...", "user": {...} }

POST /api/auth/refresh/
Body: { "refresh": "REFRESH_TOKEN" }
Returns: { "access": "NEW_JWT_TOKEN" }
```

### User (Текущий пользователь)

```
GET /api/users/current/               # Информация о текущем пользователе
POST /api/auth/register/              # Регистрация нового пользователя
POST /api/users/become-seller/        # Стать продавцом
```

### Products

```
GET /api/products/                    # Список товаров (с пагинацией)
  ?search=query                       # Поиск по названию
  &seller_id=1                        # Фильтр по продавцу
  &ordering=price                     # Сортировка

GET /api/products/{id}/               # Детали товара
POST /api/sellers/products/           # Создать товар (продавец)
PATCH /api/sellers/products/{id}/     # Обновить товар (продавец)
DELETE /api/sellers/products/{id}/    # Удалить товар (продавец)
```

### Cart

```
GET /api/cart/                        # Текущая корзина
POST /api/cart/                       # Добавить товар (product_id, quantity)
PATCH /api/cart/{item_id}/            # Изменить количество
DELETE /api/cart/{item_id}/           # Удалить товар из корзины
```

### Orders

```
GET /api/orders/                      # Мои заказы (покупателя)
GET /api/orders/my/                   # Alias для мои заказы
POST /api/orders/                     # Создать заказ
  Body: {
    "contact_name": "...",
    "contact_phone": "...",
```

### Архитектура (Feature-Sliced Design)

Проект следует **FSD** (Feature-Sliced Design) архитектуре:

**shared/** - переиспользуемый код

- `api/` - API клиент и endpoints
- `context/` - React Context (Auth, Cart)
- `lib/` - утилиты (formatPrice, formatDate)
- `constants/` - константы приложения
- `ui/` - базовые UI компоненты

**entities/** - основные сущности бизнеса

- `Product/` - компоненты товара
- `User/` - компоненты пользователя
- `Order/` - компоненты заказа

**features/** - независимые функции

- `AddToCart/` - добавление в корзину
- `Login/` - вход в систему
- `Search/` - поиск товаров

**pages/** - полные страницы

- `HomePage/` - главная страница
- `ProductsPage/` - каталог товаров
- `SellerPages/` - панель продавца (с подстраницами)

**widgets/** - сложные компоненты

- `Header/` - шапка с навигацией
- `Footer/` - подвал
- `Toast/` - всплывающие уведомления

### Именование файлов

- **Компоненты**: PascalCase (ProductCard.jsx)
- **Стили**: snake-case (ProductCard.module.css)
- **Утилиты**: camelCase (formatPrice.js)
- **Константы**: UPPER_SNAKE_CASE (API_ENDPOINTS)

## 🐛 Решение проблем

### Графики не отображаются

```bash
# 1. Проверьте, что бэкенд запущен
curl http://127.0.0.1:8000/api/sellers/analytics/

# 2. Очистите кэш браузера
Ctrl+Shift+Delete

# 3. Проверьте консоль браузера (F12)
# Ищите ошибки в Network вкладке
```

### API ошибки (401, 403, 404)

```bash
# 1. Убедитесь, что вы в системе (есть JWT токен)
# Токен хранится в localStorage

# 2. Проверьте, что вы продавец
# Используйте seller_honey / seller123

# 3. Проверьте URL endpoint
# Должно быть: http://127.0.0.1:8000/api/sellers/analytics/
```

### CORS ошибки

```python
# backend/chuvashmarket/settings.py
CORS_ALLOWED_ORIGINS = [
    "http://127.0.0.1:5173",
    "http://localhost:5173",
]
```

### Данные не обновляются

```bash
# 1. Проверьте, что в localStorage есть JWT токен
# 2. Перезагрузите страницу (F5)
# 3. Очистите кэш приложения (Ctrl+Shift+Delete)
# 4. Перезапустите dev сервер (Ctrl+C, npm run dev)
```
